# 術語表與翻譯規則

## 核心分類

| 英文 | 中文 | 說明 |
|---|---|---|
| Skill | 技能 | 可重複使用的工作指令、流程與專業方法。 |
| Plugin | 外掛／擴充包 | 用來安裝與分享技能、工具或整合設定的封裝形式。 |
| Connector | 連結器 | 讓 Claude 連接外部服務、資料與可執行動作的介接層。 |

## 常見技術詞

| 英文 | 建議中文 | 使用原則 |
|---|---|---|
| Brief | 需求簡報／任務摘要 | 行銷情境可保留 Brief，並補中文。 |
| redline | 修訂標記版 | 法務文件中可保留 redline。 |
| schema | 資料結構定義 | 開發者內容可保留 schema。 |
| Artifact | 可互動成果物 | Claude 產品名稱語境保留 Artifact。 |
| pipeline | 處理流程／管線 | 資料工程常用「管線」，一般使用者用「處理流程」。 |
| onboarding | 到職／導入流程 | 人資譯「新人到職」，產品譯「使用者導入」。 |
| enrichment | 資料補全 | 銷售名單與 CRM 語境使用。 |
| repository / repo | 程式碼儲存庫 | repo 可作為工程師慣用縮寫保留。 |
| webhook | 事件回呼 | 首次出現建議標示 webhook（事件回呼）。 |
| playbook | 作業手冊 | 依情境也可譯為策略手冊。 |
| workflow | 工作流程 | 自動化領域可保留 workflow。 |
| dashboard | 儀表板 | 台灣軟體介面常用譯法。 |

## 同名異譯規則

同一個英文名稱可能因群組與交付內容不同而採不同中文名稱。

例如 `account-research`：

- Sales：客戶研究
- Common Room：帳戶研究
- ZoomInfo：帳戶研究簡報

處理原則：

1. 英文原名永遠保留。
2. 中文名稱反映該工具實際交付內容。
3. 頁面顯示其他現有譯法。
4. 比對時以英文原名與所屬群組共同判斷。
