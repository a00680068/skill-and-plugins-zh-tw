# 翻譯品質評估

評估日期：2026-06-19

## 最終評分

### 目前使用品質：8.6／10

這份對照表已達到「一般使用者能查、能懂、能判斷何時使用」的實用水準。它不是單純逐字翻譯，而是加入功能說明、最佳使用時機、提問範例與模式建議，因此實際幫助程度高。

### 翻譯可驗證信心：7.4／10

資料模型已加入逐筆來源、版本、驗證日期與校對狀態欄位，但多數項目仍待補入單項官方來源，因此尚無法證明 650 筆皆已完成官方原文逐句核對。

## 做得好的部分

1. **英文原名完整保留**
   - 使用者可以直接回到原工具名稱搜尋。
   - 避免中文翻譯造成工具辨識困難。

2. **不是只有字面翻譯**
   - 每筆都有「能做什麼」與「最佳使用時機」。
   - 對第一次接觸 AI 技能與連結器的人特別有幫助。

3. **繁體中文一致**
   - 檢查未發現常見簡體字混入。
   - 用詞多數符合台灣使用習慣。

4. **查找效率高**
   - 可用英文、中文、描述或情境搜尋。
   - 英文名稱使用高對比色，適合快速比對翻譯。

5. **補足執行脈絡**
   - 提供 Claude Chat、Claude Code、Claude Cowork 的適配建議。
   - 提供可直接複製的結構化提問範例。

## 本輪已改善

1. **逐筆原始來源資料模型**
   - 已新增 `source_url`、`source_text`、`source_version`、`verified_at`。
   - 未查證項目明確標示「待逐筆查證」。

2. **Skill／Plugin／Connector 分類**
   - 已新增「技能」「外掛型技能」「連結器」三種資源形態。
   - 外掛型技能另行統計並使用不同識別色。

3. **情境翻譯透明化**
   - 已新增「功能導向翻譯／保留品牌原名」標記。
   - 同一英文名稱出現多個譯法時，卡片會提示所有現有譯名。

4. **技術術語說明**
   - 已建立頁面術語表。
   - 卡片偵測到常見技術詞時會顯示中文解釋。

5. **適配模式透明度**
   - 已顯示高／中／低信心。
   - 所有尚未人工確認的項目標示「待人工覆核」。

## 仍待持續改善

1. 為 650 筆資料補齊單項官方來源。
2. 建立領域審稿人與覆核流程。
3. 將「待人工覆核」逐步升級為已覆核。
4. 定期重查平台版本與功能變動。

## 已完成的校正

- 「變異分析」改為台灣財會較常用的「差異分析」。
- 修正「產財務報表」「建結構化面試計畫」等過度省略的句型。
- 「入職」調整為較符合台灣用語的「新人到職」。
- 「科學問題選擇」調整為「科學研究選題」。
- 「客訴升級」調整為「客戶案件升級處理」。
- 改善 Claude in Chrome 的台灣繁體中文說明。

## 下一輪建議

1. 每個領域抽查至少 10%。
2. 優先補齊使用量最高項目的來源。
3. 邀請領域使用者進行人工覆核。
4. 重大版本發布前重新核對平台名稱與功能。

## 最終評價

優點是實用、完整、容易比對，並且比一般純翻譯表更能幫助使用者採取行動。

缺點是來源追溯與逐筆驗證證據仍不足，因此目前適合定位為「高品質繁體中文實用對照指南」，不宜宣稱為「官方完整認證翻譯」。
