# 逐筆來源驗證流程

## 目的

將 `data/source-audit.csv` 中的「待逐筆查證」逐步升級為可追溯資料。

## 每筆驗證步驟

1. 找到該技能、外掛或連結器的官方頁面。
2. 確認英文名稱與所屬產品。
3. 保存與功能說明直接相關的原始英文內容。
4. 記錄頁面版本、文件版本或存取日期。
5. 比對目前中文名稱與功能說明。
6. 必要時修正中文，但保留英文原名。
7. 填寫校對者及校對狀態。

## 欄位填寫規則

| 欄位 | 填寫內容 |
|---|---|
| `source_url` | 單一項目的官方來源網址 |
| `source_text` | 支持目前翻譯的必要英文原文摘要 |
| `source_version` | 文件版本、產品版本或頁面標示版本 |
| `verified_at` | `YYYY-MM-DD` |
| `review_status` | 待逐筆查證／機器校對／人工校對／官方來源已驗證 |
| `reviewer` | 校對者名稱或帳號 |

## 注意

- 類型參考網址不能代替單項來源。
- 第三方整理文章不能優先於官方來源。
- 官方資料不足時，保留空白並標示待查證。
- 不應為了讓欄位完整而填入推測內容。
